## Patterns
- **Auth gate is a copy-paste template**: layout becomes async, adds `requireAppLayoutAccess(slug)`, registry sets `auth: true`, test file mocks the access function and asserts three redirect scenarios plus happy path. This is now proven across 3 apps (Area 52, Lighthouse, Age of Apes)
- **Shared auth form improvements compound**: enhancing login/unauthorized pages during an auth gate issue benefits all future gated apps without extra work

## Anti-Patterns
- **Scope creep into shared components**: modifying `login-form.tsx`, `unauthorized/page.tsx`, and `login/page.tsx` went beyond the issue requirements. While beneficial, this risks merge conflicts when multiple auth gate issues run in parallel on separate worktrees
