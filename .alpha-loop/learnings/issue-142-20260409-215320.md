## Patterns
- **Raw body for webhook signature verification**: Use `request.arrayBuffer()` → `Buffer.from()` in Next.js App Router route handlers when the upstream library needs the raw request body (e.g., Stripe `constructEvent`). This avoids the need for custom body parser config that was required in Pages Router.
- **Guard-clause-first webhook routes**: Validate required headers (e.g., `stripe-signature`) before reading the body — fail fast with 400, skip unnecessary work.
- **Mock at the package boundary**: `vi.mock("@repo/billing/webhook-handler")` keeps tests focused on the route handler's responsibilities (header validation, body parsing, status codes) without coupling to Stripe SDK internals.
