## Patterns
- Auth gate is a repeatable 3-file change: `layout.tsx` (add async + guard call), `app-registry.ts` (set `auth: true`), `__tests__/layout.test.tsx` (mock `requireAppLayoutAccess` for login/unauthorized/success paths)
- Smoke tests mock `requireAppLayoutAccess` and assert on the NEXT_REDIRECT error digest, avoiding real auth state setup while fully covering redirect behavior
- Browser verification can be safely skipped for server-side redirect gates since behavior is deterministic and fully covered by unit tests

## Anti-Patterns
- Avoid coupling unrelated UI changes (login form, unauthorized page copy tweaks) to individual auth gate PRs — these shared-component changes could cause merge conflicts across parallel auth gate branches
