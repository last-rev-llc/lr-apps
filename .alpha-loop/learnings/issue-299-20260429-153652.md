## Patterns
- Isolating a markdown renderer into its own `PlanSection` component with dedicated tests keeps the parent `ideas-app.tsx` lean and lets staleness logic (>30 day mute) be unit-tested in isolation
- Deferring UI wiring of new server actions to a follow-up issue is acceptable when the AC is render-only — call it out explicitly in code (`// TODO: wired in a later issue`) so review can confirm intentional hand-off

## Anti-Patterns
- Throwing typed errors (`RateLimitedError`, `FeatureAccessError`) from Next.js server actions without a string code/message fallback — `instanceof` does not survive the server/client boundary in production, so client toast logic must match on message or a stable error code
- Removing 8 hard-won checklist items (#147–#216 incident-derived rules) from `code-review/SKILL.md` in the same batch — these were the codebase's strongest signals against recurring regressions and should not be deleted without replacement
