## What Worked
- Migration, rollback, types, and helper delivered per AC with tests passing on first run (0 retries).
- Review caught two real CI gaps (missing auth.users seed, lint:registry silently skipping) and both were fixed in-run.
- RLS cross-tenant test harness wired into a dedicated `rls-tests` CI job with local Supabase + `db reset`.

## What Failed
- Initial RLS test seed omitted `auth.users` creation, so FK-referencing upserts would have silently failed and masked real RLS behavior.
- `lint:registry` was wired into `pnpm lint` but the `ci` job had no Supabase running, so the check exited 0 without validating anything — contradicting CLAUDE.md's claim that it fails the build.
- New "end-to-end" billing test still mocks the service-role client instead of hitting a real DB, so it doesn't deliver the integration coverage its name implies.

## Patterns
- When adding a table referenced by `auth.users(id)`, seed test users via `admin.auth.admin.createUser({ id, email, email_confirm: true })` (idempotent) before any FK-dependent inserts, and throw on upsert errors so silent seed failures can't make RLS tests pass for the wrong reason.
- Consistency-lint scripts that depend on a live DB must run in a job that actually starts that DB; gate the script to fail (not skip) when the required env is absent, or keep it off the default `lint` path.
- Layout access helper now returns `AccessResult | null`; callers with no `pathname` argument can safely use the result, but mixing a `pathname` arg with a non-null assertion is a latent footgun.

## Anti-Patterns
- Seeding FK-dependent rows without surfacing errors — tests pass because nothing got inserted, not because the policy worked.
- Wiring a "CI enforcement" script into `lint` without ensuring its runtime prerequisites exist in that CI job; silent skips give false confidence.
- Naming a test "end-to-end DB row write" while mocking the DB client — drifts test intent from test reality.
- Using `access!.user.id` with a non-null assertion at call sites that don't pass `pathname`; fine today, brittle if `publicRoutes` is later added.

## Suggested Skill Updates
- `testing-patterns`: add a "seed FK-referenced auth.users before RLS tests" note with the `admin.auth.admin.createUser` idempotent pattern, and a rule that seed upserts in test setup must throw on error.
- `code-review`: add a check for CI-enforcement scripts that silently skip when prerequisites are missing (grep for `if (!process.env.X) { console.log('skip'); process.exit(0) }` in scripts invoked by `lint`/`test`).
- `testing-patterns`: add guidance that test names containing "end-to-end" / "integration" must actually exercise the boundary they name (real DB, real HTTP, etc.) or be renamed.
