- Salvaged SSL warning panel and status-dot logic from old 313-line `health-app.tsx` while rewriting the rest into a client-grouped structure as specified.
- Comprehensive test coverage on first pass: 21 actions tests + 33 score tests + 1 page test, all passing with zero retries.
- In-memory store mock in `actions.test.ts` simulates RLS scoping (user_id filter) and unique constraint violations, catching cross-user leakage without a real DB.

## What Failed
- Sidebar dead-link missed during the move: `apps/web/app/apps/command-center/layout.tsx` MODULES still pointed at the deleted `/apps/command-center/client-health` route, producing 404s. Required a follow-up fix during review.
- `actions.ts` reads `process.env.APP_BASE_URL` and `process.env.CRON_SECRET` directly instead of going through `lib/env.ts` (the canonical pattern per CLAUDE.md).
- `lib/queries.ts` is a thin one-line re-export from `actions.ts` — satisfies the spec letter but adds indirection without value.

## Patterns
- When deleting an old route during a "move and rewrite", grep for the old path across sibling layouts/sidebars/registries — tile-on-page removal is not sufficient.
- RLS-simulating in-memory mocks (per-table user_id scoping + unique-constraint codes like 23505) give realistic coverage for Supabase server actions without integration infra.
- Group-by-client UI requires its own queries/types/score modules; don't try to reshape a sites-only data model in the component layer.

## Anti-Patterns
- Reading `process.env.*` directly inside server actions when `lib/env.ts` already declares the vars — bypasses validation and drifts from project convention.
- Creating re-export-only shim files (`queries.ts` → `actions.ts`) just to match a spec's file list; either put real query logic there or fold it back into the caller.
- Removing a feature's tile from one page and assuming the navigation is clean — sidebars/registries/breadcrumbs are independent surfaces.

## Suggested Skill Updates
- `git-workflow` or a new `route-removal` checklist: when deleting an app route, grep for the route path in `layout.tsx`, sidebar/MODULES arrays, `app-registry.ts`, and breadcrumb configs before declaring the move complete.
- `code-review`: add a "dead link sweep" check for PRs that delete or move routes — verify no remaining `href`/`Link` references to the old path.
