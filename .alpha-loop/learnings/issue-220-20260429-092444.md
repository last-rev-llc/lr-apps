## What Failed

## Patterns
- When multiple sibling handlers in a component share a UX contract (e.g., optimistic update + rollback), apply the pattern uniformly across all of them — review each handler against the same checklist.
- Server-side analytics `capture(userSub, event, props)` from a server-only callback is equivalent to client-side `track()` for PostHog ingestion when distinct_id is set explicitly; prefer server-side when the trigger point is server-side anyway.
- Layout tests can omit `@repo/analytics/server` mocks because `capture()` no-ops under `NODE_ENV=='test'` — relying on env-based no-op is cleaner than per-test mocking.

## Anti-Patterns
- Implementing the same UX requirement (optimistic UI) for 3 of 4 handlers and silently skipping the 4th — verify AC coverage handler-by-handler, not at the component level.
- Skipping `pnpm typecheck` on changed files before declaring done; tests passing does not imply types compile.
- Optimistic UI without rollback on failure (handleGlobal/handleTier surface error but don't revert) — acceptable here per AC silence, but worth flagging as a half-finished pattern.
