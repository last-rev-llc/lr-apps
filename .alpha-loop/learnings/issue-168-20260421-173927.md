## Patterns
- When mocking typed rows (e.g. `SubscriptionRow`), grep the actual type definition rather than guessing field names — vitest won't fail on shape mismatches but `tsc --noEmit` will.
- Edge-runtime constraints matter: `@opentelemetry/sdk-node` cannot load on edge, so any proxy.ts spans are no-ops. Document the runtime split explicitly when adding observability.

## Anti-Patterns
- Bundling unrelated infra changes (Sentry gitignore, Upstash env, OTel env, CONTRIBUTING.md) into a docs-only "architecture diagram" issue. Each belongs in its own issue/PR for reviewability and revert-safety.
- Shipping cache write paths without a corresponding read path. If the read API exists but is unused, either wire it up or delete the writes — write-only caches are pure cost.
- Adding env vars to docs without also adding them to `turbo.json` `globalEnv` and `.env.example` (the project's own non-negotiable, per CLAUDE.md).
