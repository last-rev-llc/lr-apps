## What Worked
- RLS test harness with `withAuthContext` helper successfully implemented using `SET LOCAL role` + `request.jwt.claims` for per-user query context.
- Tests properly skipped via `describe.skipIf(!process.env.SUPABASE_TEST_URL)` so local `pnpm test` passes without requiring Supabase running.
- Reviewer caught and fixed two real CI gaps (missing auth.users seeding, lint:registry silently no-op'ing) before merge.

## What Failed
- Initial implementation seeded `app_permissions`/`subscriptions`/`audit_log` without first creating users in `auth.users`, which would have caused FK violations in CI and silent test pass-for-wrong-reason.
- `lint:registry` was wired into the `ci` job (no Supabase) instead of the `rls-tests` job, making the registry→DB consistency check a no-op in CI.
- Three layouts use `access!.user.id` non-null assertion which is brittle if `publicRoutes` is later added to those apps.

## Patterns
- For RLS integration tests: always seed `auth.users` first via `admin.auth.admin.createUser({id, email, email_confirm: true})` before inserting into FK-referencing tables; make seed calls idempotent and surface errors via thrown exceptions.
- Gate environment-dependent tests with `describe.skipIf(!process.env.X)` and run them via a dedicated script (`test:rls`) so default `pnpm test` stays hermetic.
- CI checks that depend on a running service (Supabase) must live in the job that starts that service, not in a generic `lint` job.

## Anti-Patterns
- Silent skip-on-missing-env in lint scripts: a check that exits 0 when its prerequisite is unset provides no protection in CI; either fail loudly or wire it into the job where the prerequisite exists.
- Seeding rows without surfacing errors (`.upsert(...)` without `.throwOnError()` or error checking) — masks FK violations and causes downstream tests to pass for the wrong reason (zero rows ≠ RLS denial).
- Non-null assertions (`access!.user.id`) on helpers whose null-return contract depends on caller arguments — fragile when the call site evolves.

## Suggested Skill Updates
- `testing-patterns`: add a section on RLS/integration test seeding requiring `auth.users` rows before any FK-dependent inserts, with idempotent `createUser` pattern and the "zero rows ≠ denial" failure mode.
- `code-review`: add a check for "CI scripts that silently skip when env vars are unset" — verify the script is wired into a job where its prerequisites are guaranteed present.
