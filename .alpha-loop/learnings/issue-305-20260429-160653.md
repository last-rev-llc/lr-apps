## Patterns

- Accessible-name calculation: when a button's subtree contains only emoji/icon, `title` is overridden — add explicit `aria-label` so `getByRole({ name })` resolves.
- Use `.last()` (or scoped container) on Playwright role queries when sibling lists (e.g., Categories PillList + filter bar) can render duplicate accessible names.

## Anti-Patterns
- Relying on `title` attribute for Playwright `getByRole({ name })` matches when the button content is an emoji — accessible name comes from subtree text, not title.
- Unscoped `getByRole('button', { name: /^all$/i })` in pages with both a Categories pill list and a show-filter — strict mode flags the duplicate.
