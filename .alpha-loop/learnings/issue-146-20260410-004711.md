## Patterns
- **Fail-open idempotency**: Record processed event IDs but don't block webhook processing if the DB write fails — downstream operations (upsertSubscription) are themselves idempotent, so the duplicate check is an optimization not a correctness requirement
- **Price consistency check**: Seed scripts that create Stripe objects should cross-reference in-app tier config to avoid charge/display mismatches
- **TOCTOU acceptance**: Document known race conditions with rationale for why they're safe rather than adding distributed locks for marginal gains

## Anti-Patterns
- Hardcoding prices in seed scripts without referencing the canonical tier config — leads to silent mismatches between what users see and what Stripe charges
- Documenting env vars for tiers that don't use them (free tier has no Stripe price) — causes confusion during setup
