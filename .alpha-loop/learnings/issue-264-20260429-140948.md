## Patterns
- Smoke-test issues for routing changes pair well with proxy integration tests (`__tests__/proxy.integration.test.ts`) that assert host classification + redirect behavior across clusters.
- When documenting an API surface, distinguish exported helpers from internal helpers explicitly — readers will try to import anything listed as a callable.
- `lib-listing:start`/`lib-listing:end` markers enforced by `scripts/check-claude-md-lib-sync.ts` are a robust way to keep doc-as-source-of-truth blocks from drifting.

## Anti-Patterns
- Listing internal/private functions in a "public API" docs section invites misuse — always mark scope (exported vs. internal) when the file actually controls visibility.
- CLAUDE.md previously contained meta-commentary about a prior rewrite instead of actual instructions — a doc that describes itself rather than the system is a smell.
