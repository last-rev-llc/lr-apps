## Patterns
- Assert the **contract** (db helpers *throw* on Supabase error so error.tsx can catch) rather than asserting UI output — keeps tests fast and meaningful
- For large registries, combine narrow-deep tests (3 sampled apps, full middleware chain) with broad-shallow tests (every app, pure registry logic) instead of N×full-integration
- Mock `getAuth0ClientForHost` at the module boundary and use `NextResponse` helpers directly so tests verify real Next.js middleware headers (`x-middleware-rewrite`, `x-middleware-next`, `set-cookie`)

## Anti-Patterns
- Don't assert AC text literally when implementation diverges ("404" vs 3xx redirect) — test actual behavior and call out the AC divergence in review, don't force a wrong assertion
- Avoid iterating every registry entry through a full middleware mock — sample + separate pure-function coverage is cheaper and equally sound
