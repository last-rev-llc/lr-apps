## Patterns
- Hardcoded hex colors in email templates are the documented exception to the no-hardcoded-colors rule (email clients strip `<link>`/`<style>`, require inline styles) — call this out in the file header comment so reviewers don't flag it.
- Ship shared packages (`@repo/email`, `@repo/storage`) standalone with their own tests even when no caller wires them up yet; downstream issues consume them later.
- `withRequestContext` + structured `log.*` calls in API routes produce traceable JSON logs in test output, making failures easy to attribute.

## Anti-Patterns
- Don't write preview HTML output into a tracked directory — always gitignore the preview output path when adding a preview/test script.
