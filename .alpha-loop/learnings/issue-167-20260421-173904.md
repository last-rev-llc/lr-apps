## Patterns
- When generating test mocks for typed fixtures, run `tsc --noEmit` in addition to vitest — runtime-passing tests can still break the typecheck gate.
- Scaffolding scripts that touch the registry should also verify the generated entry loads (the `app-registry.test.ts` uniqueness suite doubles as a smoke test).

## Anti-Patterns
- Adding `cacheSet` calls without a corresponding `cacheGet` reader — produces write-only caches that waste ops and latency. If async cached variants exist, either wire them in or remove the writes.
- Declaring a workspace in `vitest.workspace.ts` without adding a package.json + test script — turbo silently skips it and tests never run in CI.
- Documenting an env var in ops docs without also adding it to `turbo.json` globalEnv — Turbo won't propagate it to child tasks at build/run time.
