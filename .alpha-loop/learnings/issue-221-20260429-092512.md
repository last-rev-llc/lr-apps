## Patterns
- Safe no-op degradation: PostHog client returns `[]` when env vars unset and skips fetch entirely — keeps the page renderable in dev/test without API keys
- Hashing PII at the data-access layer (`distinct_id` → sha256 `userIdHash`) before it reaches the UI keeps user identifiers out of admin-visible event lists
- Server-side `capture(userSub, ...)` substituted for client-side `track(...)` on Auth0 callback because the callback executes server-side; PostHog receives equivalent events

## Anti-Patterns
- Inconsistent optimistic-update coverage across sibling handlers (3 of 4 mutation handlers had optimistic UI, one didn't) — when adding optimistic updates, audit all peer handlers in the same component
- Test mock builders that infer `unknown` from chained Supabase-style fluent APIs need explicit type narrowing at the call site or `pnpm typecheck` fails on changed files
