## Patterns
- When moving feature directories between app namespaces, deleting the old `lib/queries.ts` simultaneously with creating the new one in the same diff keeps the change atomic and avoids stale-import windows
- Co-locating tests under `__tests__/` adjacent to the moved code (e.g. `app/apps/ideas/__tests__/actions.test.ts`) preserves the existing repo convention used by every other app under `apps/web/app/apps/*`

## Anti-Patterns
- Carrying over dead code during a file move (the unused `SOURCE_COLORS` const flagged by lint was migrated verbatim from the legacy file) — a move is the cheapest moment to drop unused exports, since there are no external callers to worry about
- Migration filename drift between issue spec (`20260429_ideas.up.sql`) and actual repo convention (`20260429_ideas.sql`) — issue authors should check `supabase/migrations/` naming before specifying exact filenames
