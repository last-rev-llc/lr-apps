## Patterns
- When two server actions can produce the same end state (status dropdown reaching `archived` vs. dedicated `archiveIdea`), assert the side-effect contract (`completedAt` handling) in tests for **both** entry points so they cannot drift.
- Hoisting test UUIDs via `vi.hoisted` keeps mock auth user IDs consistent between `vi.mock` factories and assertions.

## Anti-Patterns
- Shipping a feature-label constant (`FEATURE_LABELS`) with no reader while the real gating lives in a parallel map: creates two sources of truth for tier→feature mapping. Either wire a consumer or delete the export.
- Exposing the same state transition through multiple actions without a shared helper risks one path silently violating an AC the other enforces (the `completedAt`-on-archive bug).
