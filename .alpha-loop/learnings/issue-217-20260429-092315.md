## Patterns
- Thin backend-swap wrappers for third-party SDKs: stable `track()`/`capture()` interface, backend isolated to one file, env-driven no-op fallback when keys are unset.
- Default-no-op-in-test analytics keeps consumer test files from needing per-suite mocks (verified: layout tests pass without mocking `@repo/analytics/server`).
- Aggregate review findings inline with severity tags (`warning`/`info`) and explicit fixed/unfixed status — makes the trade-offs auditable.

## Anti-Patterns
- Inconsistent optimistic-update handling across sibling handlers in the same component (3 of 4 handlers updated state, 1 didn't) — should be enforced uniformly when adding the pattern.
- Silently translating a client-side AC to server-side without flagging in the issue/PR — equivalent behavior is fine, but the deviation should be surfaced during planning, not at review time.
