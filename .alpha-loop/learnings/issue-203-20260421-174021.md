## Patterns
- Cache-aside with graceful degradation: check env vars → return direct query if absent; wrap Redis calls in try/catch so cache failures never break the hot path.
- Version-prefixed keys (`CACHE_VERSION` or `VERCEL_GIT_COMMIT_SHA`) enable zero-touch invalidation on deploy for "indefinite" static-data caches.
- Colocate invalidation with mutation: `upsertPermission()` and Stripe webhook upsert call `cacheDelete` for the affected key, keeping write paths authoritative.

## Anti-Patterns
- Write-only cache: `getAppBySubdomain`/`getAppBySlug` fire `cacheSet` on every call but no caller imports the async `*Cached` variants that would read. Burns Upstash ops with zero hit-rate benefit. If an async variant has no reader, don't write either — wire it end-to-end or delete it.
- Running `vitest run` only and calling it done: skipping `tsc --noEmit` hid the `plan`/`tier` type error. Typecheck must be part of the done-definition for any new test file with typed mocks.
- Orphaned test suites: `scripts/__tests__/*.test.ts` exist but aren't wired into turbo because `scripts/` has no `package.json` with a test script. Tests that don't run are worse than no tests.

## Suggested Skill Updates
- `testing-patterns`: add a note that new test files must be validated with both `vitest run` AND `pnpm typecheck` before declaring done — vitest does not enforce strict TS on mock shapes, so type drift in test fixtures (e.g. `plan` vs `tier`) passes runtime but fails CI typecheck.
- `code-review`: add a check for "write-only cache" / dead-write patterns — if a `cacheSet` has no corresponding `cacheGet` caller, flag it as waste rather than future-proofing.
