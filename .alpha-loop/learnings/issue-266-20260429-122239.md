## Patterns
- When a migration plan references "same pattern as X" but X doesn't exist yet, define the shared primitive in the current migration with a comment establishing the reuse contract for future migrations.
- Bumping a registry-count test assertion (`>= 27` → `>= 28`) in the same commit that adds the app keeps `CLAUDE.md`, the registry, and the integrity test in sync atomically.
- Quoted camelCase column names (`"contractStatus"`, `"updatedAt"`) used consistently with check constraints referencing the same quoted form — avoids Postgres lowercasing surprises.

## Anti-Patterns
- `clients.down.sql` drops `public.set_updated_at()` even though sibling migrations (`client_sites`, etc.) reuse it — reverse-order rollback works, but a partial/single-migration rollback would fail. Shared trigger functions should be owned by a dedicated migration or guarded with a dependency check, not dropped by the first user's down-migration.
