## Patterns
- **App test trinity**: each app gets three test files (layout, page, app-component) covering auth gates, data fetching, and UI rendering respectively
- **Mock-at-boundary pattern**: mocking `requireAppLayoutAccess` and Supabase queries at module boundaries rather than deep internals keeps tests resilient to refactors
- **Tab interaction via `fireEvent.keyDown` + Enter**: used for Radix-style tabs where `fireEvent.click` on tab triggers don't reliably update panels
