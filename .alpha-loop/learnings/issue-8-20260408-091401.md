## Patterns
- **Auth redirect builder as a testable pure function**: Extracting `buildAuthLoginHref` into its own module (`auth-login-redirect.ts`) made redirect logic independently testable with 7 unit tests covering edge cases (expired sessions, invalid redirects, unknown apps)
- **Layout test pattern via module re-import**: Using `await import("../layout")` to call the default export directly, with `requireAppLayoutAccess` mocked, provides clean coverage of auth gate behavior without rendering JSX

## Anti-Patterns
- **Duplicated subdomain-to-slug mapping**: `SUBDOMAIN_TO_SLUG` in `self-enroll.ts` duplicates data already in `app-registry.ts`, creating a sync risk. The review flagged this but it was left unfixed — should extract to a shared utility
- **Open redirect via protocol-relative URLs**: `isSafeReturnTo` only checked for leading `/` without rejecting `//`, which `new URL` resolves as a protocol-relative redirect to an external domain. Always reject `//` in return-to validation
