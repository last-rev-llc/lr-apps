- Per-column iteration in the smoke test names the failing column in the assertion message, making PostgREST `42703` failures triagable instead of opaque.
- `it.skipIf(!hasCreds)` lets the canary run in CI (where Supabase env is set) without breaking local `pnpm test` runs missing credentials.

## Patterns
- **Schema-drift canary for externally-owned tables**: pin column list as a const, import everywhere, write a per-column smoke test that asserts each pinned column exists upstream and names it on failure.
- **Doc the cross-repo coupling next to the pin site**: a CLAUDE.md "External Schema Dependencies" section pointing readers to the const + canary makes the contract discoverable.
- **Conditional CI-only test via `it.skipIf`**: env-gated integration tests that skip cleanly without creds run in CI without forcing local devs to provision secrets.

## Anti-Patterns
- Acceptance criteria naming a specific file (`lib/queries.ts`) when that file is a re-export facade — review surfaced the pin actually lives in `actions.ts`. Either update the AC to name the real call site or relocate the select to match the AC, otherwise reviewers chase ghosts.
- Switching from `select('*')` to a pinned list without auditing every renderer for fields that may have been read implicitly (the `uptime` page's `site.history` access is now silently always empty).

## Suggested Skill Updates
- None — pattern is too narrow (externally-owned Supabase tables) to generalize into an existing skill. If this recurs across more cross-repo tables, consider a small "external-schema-canary" pattern note in `testing-patterns`.
