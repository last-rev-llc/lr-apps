## Patterns
- Use `glass-sm` utility class as the standard replacement for inline `border-white/* bg-white/* backdrop-blur-sm` card styling in dark-themed apps
- `Button asChild` pattern works well for wrapping `Link` components while preserving custom gradient styles
- Intentional near-white emphasis (`text-white/80`, `text-white/90`) on primary content is acceptable and distinct from `text-muted-foreground` — don't over-normalize

## Anti-Patterns
- Don't migrate form elements (inputs, selects) to `@repo/ui` when no matching component exists in the shared library — leave inline styles rather than forcing a partial abstraction
