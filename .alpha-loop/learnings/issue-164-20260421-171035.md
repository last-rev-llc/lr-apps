## Patterns
- Double-submit CSRF: cookie (readable by JS, SameSite=Lax, Secure in prod) + matching x-csrf-token header; verify equality on state-changing methods
- Webhook exemption list for CSRF skip (stripe signature is the auth mechanism, not CSRF token)
- Reset helper exports (`_resetRateLimitStore`) for in-memory stores so integration tests are deterministic across describe blocks

## Anti-Patterns
- In-memory rate limit store on serverless = N × configured limit per instance; ship with a TODO to migrate to Redis/Upstash before publishing SLA figures
- Accepting unused parameters (`buildCspOptions.reportOnly`) — either wire them up or remove to avoid dead API surface
- Inline Zod validation in routes that have a shared validateJson helper creates inconsistency; document the exception (sendBeacon body-less 204) or always use the helper
