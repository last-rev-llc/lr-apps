## Patterns
- When a scaffolded app's registry entry changes the total count, also bump the `>=N` assertion in `app-registry.test.ts` and the prose count in `CLAUDE.md` in the same change to avoid a follow-up lint/test break.
- Introducing a shared SQL helper (e.g. `public.set_updated_at()`) inside the first migration that needs it, with an inline comment naming subsequent reusers, keeps the function's lifecycle traceable without a dedicated bootstrap migration.

## Anti-Patterns
- Don't reference a "same pattern as ideas" precedent in plans without verifying that precedent exists in the repo — the issue plan cited an ideas migration that isn't present, which could mislead future authors copying the pattern.
- Avoid down-migrations that drop a shared helper function without acknowledging the cross-table dependency; ordering saves correctness here, but the coupling deserves a comment so a partial rollback isn't surprising.
