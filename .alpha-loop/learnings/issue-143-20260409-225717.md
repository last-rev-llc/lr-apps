## What Failed

## Patterns
- Code review caught wiring bugs (wrong redirect URLs) that unit tests couldn't — route handler tests mock Stripe and only verify the shape of params, not that destination pages exist
- Creating both `/checkout/success` and `/checkout/cancel` pages alongside the checkout flow prevents orphaned redirect targets
- Environment variables (`STRIPE_PRICE_ID_PRO`, `STRIPE_PRICE_ID_ENTERPRISE`, `APP_BASE_URL`) added to both `.env.local` and `turbo.json globalEnv` in the same pass — avoids the common "works locally, breaks in CI" gap

## Anti-Patterns
- Hardcoding redirect paths in route handlers without verifying the target routes exist — easy to typo (`/my-apps` vs `/apps`) and tests won't catch it
- Setting `cancel_url` to a different page than the dedicated cancel route creates dead code that looks intentional but is never reachable
