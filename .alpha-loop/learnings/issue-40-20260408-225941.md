## Patterns
- When migrating to shared components that render non-interactive elements (e.g., Badge renders a `<div>`), always wrap in a `<button>` if the original element was interactive. Check the rendered DOM element of each shared component before substituting.
- Adding `type="button"` to non-submit buttons during migration is a low-cost a11y improvement worth including systematically.

## Anti-Patterns
- Replacing `<button>` with a component that renders a `<div>` without checking — silently breaks keyboard/screen-reader accessibility with no test failure to catch it.
