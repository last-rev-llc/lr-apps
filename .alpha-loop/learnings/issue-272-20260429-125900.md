## Patterns
- Mock RLS by scoping every `from(table)` query through a `currentUserId` filter on `user_id` columns — lets a single test toggle `currentUserId = "user-b"` to verify cross-user reads return zero rows
- Mirror DB defaults (openTicketCount=0, isPrimary=false) in mock insert so cross-user assertions don't see undefined and false-positive
- Simulate Postgres unique-constraint violations by returning `{ error: { code: "23505" } }` from `.insert().select().single()` to test friendly-error mapping
- Thin `lib/queries.ts` re-export of read actions from `actions.ts` satisfies the spec's "queries file" requirement without duplicating logic

## Anti-Patterns
- Reading `process.env.APP_BASE_URL` / `process.env.CRON_SECRET` directly inside server actions instead of going through `lib/env.ts` — CLAUDE.md flags `lib/env.ts` as canonical; existing apps already drift from this (cringe-rizzler, cc-analytics)
- Sidebar/registry entries pointing at deleted route groups — issue #274 removed a tile but left a `MODULES` entry in `command-center/layout.tsx`, producing a runtime 404. Removing a route requires sweeping `MODULES`, sidebar configs, and matching layout test assertions
