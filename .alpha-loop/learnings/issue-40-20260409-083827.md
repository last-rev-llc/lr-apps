## Patterns
- When a shared UI component renders a block element (e.g., Badge → `<div>`), use `role="button"` + `tabIndex` + `onKeyDown` directly on it instead of wrapping in `<button>` to avoid invalid HTML nesting
- Component migrations that touch only markup (not logic/state) can pass tests on first attempt when the shared components have matching APIs

## Anti-Patterns
- Wrapping block-level shared components inside `<button>` elements without checking the rendered HTML structure — produces invalid DOM
- Leaving rating buttons as plain `<button>` when all other interactive elements were migrated to shared components (minor inconsistency noted but not required by AC)
