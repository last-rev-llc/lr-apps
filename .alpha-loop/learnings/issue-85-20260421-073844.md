## Patterns
- Assert on Next.js middleware response headers (`x-middleware-rewrite`, `x-middleware-next`, `x-middleware-override-headers`) to verify rewrite behavior without executing the full request pipeline.
- Use `vi.stubEnv("NODE_ENV", "development")` with `afterEach(vi.unstubAllEnvs)` to test dev-only routing branches (e.g., `?app=<slug>` query param).
- Flag AC-vs-implementation divergence as `[info]` review items rather than test defects when tests correctly reflect actual behavior (e.g., 3xx redirect vs. literal 404).

## Anti-Patterns
- Don't literally test "returns 404" when implementation actually redirects — assert on observable behavior (status range 300-399, absence of rewrite header) instead.
- Avoid iterating all 27+ apps in middleware integration tests when registry-level tests already provide that coverage; duplicates slow the suite without adding signal.
