## Patterns
- New shared packages in this monorepo follow a consistent shape: `packages/<name>/src/` with a single public entry, co-located `*.test.ts` using `vi.mock` for the external client, and `vitest run` as the test script.
- Storage helper pattern: thin wrapper around provider SDK exposing upload/download/delete/list + signed URL, with size/type validation at the boundary — keeps callers provider-agnostic.
- Shipping a package before any caller consumes it is acceptable here when the acceptance criteria explicitly scope the deliverable to the package itself (downstream wiring is a separate concern).

## Anti-Patterns
- Don't block a package-scoped issue on finding a consumer — if AC says "helper in `@repo/storage`", ship the helper with tests and let integration issues wire it in later.
