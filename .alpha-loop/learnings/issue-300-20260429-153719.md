## Patterns
- When removing a tile from a dashboard module list, derive the count from `MODULES.length` rather than hardcoding — prevents future drift on add/remove.
- For app-promotion (sub-route → standalone subdomain), pair the local-route removal with a 301 redirect at the proxy layer so existing bookmarks/links don't 404. Test both clusters (apps.lastrev.com and lastrev.com) explicitly.
- Update the corresponding layout test's "Hub + N module items (N+1 total)" assertion to a count-agnostic version when modules churn frequently.

## Anti-Patterns
- Hardcoded count literals in headers ("21 Routes") that drift when the underlying list mutates — recurring class of bug across this codebase.
- Removing a sub-route without adding a redirect — leaves bookmarked URLs pointing at a removed page.
