## Patterns
- Analytics calls degrade to safe no-ops when env unset / `NODE_ENV==='test'`, so existing layout tests don't need to mock `@repo/analytics/server`.
- SHA-256 hash user IDs (`distinct_id`) before storing/displaying to satisfy "no PII" requirement while keeping correlation.
- Document every event in package README event catalogue at the same time as instrumenting it, to prevent analytics sprawl.

## Anti-Patterns
- Inconsistent optimistic UI across sibling handlers (3 of 4 had it, 1 didn't) — when adding optimistic updates, audit all related handlers in one pass.
- Loose `unknown` typing in test mock builders triggers `TS2571`; narrow at the call site rather than relying on inference.
