## Patterns
- **Hoisted mock builder for Supabase**: `vi.hoisted()` to create a chainable mock builder (`select → eq → order → single`) that can be shared across tests and reset in `beforeEach` — reusable for any Supabase-backed component
- **Separate test files by layer**: `queries.test.ts` for data-fetching logic, `joke-viewer.test.tsx` for interactive component, `page.test.tsx` for server page composition — mirrors the component/query split in the app code
- **Category filtering via click + assert**: testing filter badges by clicking a category and asserting the filtered joke appears, plus verifying pool count text (`"1 joke in Animals"`)

## Anti-Patterns
- **Seed formula coupling**: `queries.test.ts` hardcodes the JOTD seed formula (`year*10000 + month*100 + day`); if the algorithm changes, the test breaks silently with wrong assertions rather than a clear failure. Consider testing the contract (deterministic, date-based) rather than the formula.
- **`any` types in mock props**: mock component signatures use `any` for props — acceptable in tests but can mask type regressions if the real component's prop interface changes
