## Patterns
- Keep registry fields as minimal canonical tokens (leftmost label) and derive composite values (full host, origin, auth hub) via a single helper module — makes cluster cutovers (apps.lastrev.com vs. lastrev.com) a one-file change.
- Add a structural invariant test (`subdomain.includes(".") === false`) when documenting a "this field means X, not Y" decision — prevents future drift even without a schema change.
- Co-locate cluster predicates (`isAppsClusterHost`, `isLegacyClusterHost`) with the host builder so proxy redirects and URL construction share one source of truth.

## Anti-Patterns
- Don't reference test files by guessed path in acceptance criteria (issue cited `proxy.test.ts`, actual is `proxy.integration.test.ts`) — verify filenames before writing the AC.
- Don't bundle pre-existing typecheck noise (e.g. `vi.mockResolvedValueOnce` typing in `uptime/page.test.tsx`) into a scoped issue; leave it for a dedicated fix.
