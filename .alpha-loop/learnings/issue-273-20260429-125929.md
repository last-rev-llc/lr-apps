## What Worked
- Pure-function design with locked `WEIGHTS` constant exported alongside `computeHealthScore` made table-driven tests trivial — 33 score tests passed first try with zero retries.
- Per-signal breakdown returned as `Record<keyof typeof WEIGHTS, number>` satisfied both the UI tooltip requirement and gave tests a clean per-axis assertion target.
- SSL threshold logic (`<7d` critical, `<30d` warn, expired) and contract tiers (`active`/`expiring-soon`/`expired`/`none=neutral`) encoded as explicit branches kept the function deterministic and easy to cover.

## What Failed
- Nothing on the score helper itself. Reviewer flagged a pre-existing dead-link bug in command-center sidebar (unrelated to #273) and fixed it as a drive-by.

## Patterns
- For weighted-scoring helpers: export the weights table as `as const`, type the breakdown via `keyof typeof WEIGHTS`, and write one test row per (axis × {perfect, mid, broken, missing-data}).
- Keep scoring pure (no I/O, no logging, no clock reads — pass `now` in if needed) so tests are deterministic without mocks.

## Anti-Patterns
- Don't bury the weights inside the function body — exporting them lets callers (and the UI tooltip) reference the same source of truth and prevents silent drift.
- Don't co-locate fetch/query logic with scoring; the spec called for `lib/score.ts` to be pure, and mixing in I/O would have broken the table-driven test pattern.
