## Patterns
- Vitest does not strict-typecheck mocks; rely on `tsc --noEmit` (or a CI typecheck step) to catch drift between test fixtures and the real type — runtime-green ≠ type-clean.
- For OTel in Next.js, accept that edge-runtime middleware spans are no-ops and instead anchor traces in nodejs-runtime layout/route handlers (e.g. the `requireAppLayoutAccess` permission_lookup span).
- Any new env var must land in three places together: `.env.example`, `turbo.json` globalEnv, and (if user-facing) `docs/ops/*.md` — partial wiring leaks at runtime, not at build.

## Anti-Patterns
- Adding write-only cache calls "for future use" — pays the latency/cost now with zero current benefit. Wire the read path in the same change or omit the writes.
- Documenting a config variable (`OTEL_EXPORTER_OTLP_HEADERS`) without adding it to `turbo.json` globalEnv — docs imply it works, Turbo silently drops it.
- Placing tests in `scripts/__tests__/` without a per-workspace `package.json` test script — vitest.workspace.ts lists the dir, but turbo never invokes it, so the suite is invisible to `pnpm test`.
