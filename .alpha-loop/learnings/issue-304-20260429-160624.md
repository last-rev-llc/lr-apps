## What Worked
- All 1030 unit tests passed across 12 packages with full turbo cache hits.
- Playwright e2e spec covers the full flow (sign-in → create → edit → rate → snooze → archive → RLS check) and review caught two locator bugs before merge.
- Adding `aria-label="Snooze"` to the icon-only button both fixed the locator and improved accessibility.

## What Failed
- Initial `getByRole('button', { name: /^snooze$/i })` locator did not match because the snooze button is icon-only (`⏰`) — `title` attribute does not contribute to accessible name when subtree text exists.
- `getByRole('button', { name: /^all$/i })` matched two elements (Categories PillList + show-filter) and threw under Playwright strict mode.
- AC #4 ("AI stubbed at network layer") could not be fully met: Plan & score button is still a TODO in `ideas-app.tsx`, so the test seeds a pre-populated plan via service-role helper instead — documented as scope limitation.

## Patterns
- For icon-only buttons that need e2e locators, use `aria-label` rather than `title` — Playwright `getByRole` follows W3C accessible-name calculation where subtree text beats `title`.
- When a page renders multiple elements with the same accessible name (e.g., "All" pill and "All" filter), scope locators with `.last()` / `.first()` or a parent container selector to satisfy strict mode.
- Seeding test data via service-role Supabase client is a reasonable substitute for network-layer stubs when the calling UI is still a TODO.

## Anti-Patterns
- Relying on `title` for accessible name on icon-only buttons — fails both Playwright role queries and screen readers.
- Writing role-based locators without scoping when ambiguous text ("All", "Save", "Cancel") appears in multiple places on the same page.
- Treating "AC technically met by audit script" as sufficient when sibling files in the same directory still carry the patterns the AC was meant to eradicate (issue #302 cleanup of `text-white/N` only hit `ideas-app.tsx`, leaving 4 sibling components untouched).

## Suggested Skill Updates
- `test-robustness`: add a section on Playwright accessible-name pitfalls — icon-only buttons need `aria-label`, ambiguous labels need `.first()`/`.last()`/parent scoping under strict mode.
- `code-review`: add a check for "AC scope coverage across sibling files" — when an AC says "audit passes for X/**", confirm the cleanup applied to all files in the glob, not just the file named in the issue title.
