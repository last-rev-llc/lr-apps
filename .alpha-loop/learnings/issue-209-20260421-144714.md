## Patterns
- When adding Zod env validation, widen schema source to `Record<string, string | undefined>` rather than casting to `NodeJS.ProcessEnv` — TS 5 requires NODE_ENV and breaks test fixtures
- Ops runbooks benefit from a machine-readable pointer in `.alpha-loop.yaml` (`ops_documentation:` section) so future agents surface procedures without grepping

## Anti-Patterns
- Claiming behavior in docs that the code doesn't actually perform (e.g. "validates at startup" when `env()` is never called) — ship the wiring or soften the claim
- Placing test files under `scripts/__tests__/` without a package.json — they silently drop out of `turbo run test` and only run if invoked directly
