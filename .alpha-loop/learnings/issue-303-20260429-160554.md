## Patterns
- For server-action RLS coverage without spinning up real Postgres: drive the supabase client mock to return `{ error: 'not found' }` on cross-user `.eq('user_id', ...)` paths and assert the action surfaces the same error shape — proves user_id scoping without needing real RLS.
- AI-action tests should toggle the local-dev stub by deleting `process.env.ANTHROPIC_API_KEY` in a `beforeEach`, asserting the warn-log "ANTHROPIC_API_KEY unset — using deterministic stub" fires.
- Rate-limit tests: invoke the action N times in a loop, assert the (N+1)th throws `RateLimitedError` and the underlying model fn was called exactly N times.

## Anti-Patterns
- Don't rely on `title` attribute for Playwright `getByRole({ name })` — subtree text (e.g. emoji "⏰") wins per ARIA accessible-name calculation. Use `aria-label`.
- Don't use unscoped `getByRole('button', { name: /^all$/i })` when multiple "All" buttons exist (filter pill + category pill); strict mode fails. Scope with `.last()`, parent locator, or distinct `aria-label`.
- `audit-tokens.ts` only flags Tailwind palette classes and hex/rgb/oklch literals — `text-white/<n>` opacity utilities and `var(--color-X)NN` inline styles slip through. Don't treat a 0-violation audit as proof of full token compliance.
