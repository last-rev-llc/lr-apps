## Patterns
- Marker-comment pattern for PR bot comments (`<!-- preview-app-links -->` sentinel + `listComments` → find → update-or-create) avoids duplicate comments on re-deploys.
- Widening function parameter types (`Record<string, string | undefined>`) is preferable to casting test fixtures to `NodeJS.ProcessEnv` under stricter TS ProcessEnv typings.
- Pairing migration lint (`pnpm db:check-migration-pairs`) as a dedicated CI step **before** `pnpm lint` surfaces missing `.down.sql` files early with a clearer failure message.

## Anti-Patterns
- Claiming runtime validation happens "at process startup" when the validator is only exported (not invoked) — docs drift from behavior. Either wire it in or describe it as a library.
- Documenting a workflow trigger that doesn't match the YAML (`pull_request` vs `deployment_status`) — docs should be generated from or cross-checked against the actual workflow file.
- Using `as NodeJS.ProcessEnv` to coerce partial env fixtures; breaks under stricter TS lib typings that require `NODE_ENV`.
