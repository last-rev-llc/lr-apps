## What Failed

## Patterns
- When replacing inline styled elements with shared components, grep tests for the old class names immediately — badge/color migrations always break snapshot-style class assertions
- Code review after migration is essential: the agent missed 4 bare `div.glass-sm` wrappers on first pass that the reviewer caught
- Mapping domain values to component variants (e.g., `moodToVariant` record) is a clean pattern for bridging app-specific semantics to shared component APIs

## Anti-Patterns
- Making tangential changes outside the issue scope (command-center emoji wrapping, meeting-summaries capitalization fix) — increases review surface and risk for no benefit
- Replacing theme token classes with hardcoded Tailwind equivalents during migration — always preserve existing token usage unless the migration explicitly requires changing it
