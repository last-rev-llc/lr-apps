## Patterns
- When two code paths can reach the same state transition (archive via row menu vs. archive via status dropdown), assert the same contract in both — the cross-path consistency check found a silent data-loss bug.
- Spec-mandated config entries (tier-config, feature flags) are low-risk, single-line changes that benefit from cross-checking against the actual gate consumer (`@repo/billing` `FEATURE_TIER`) to flag duplicate sources of truth.

## Anti-Patterns
- Adding a parallel feature-label map (`FEATURE_LABELS`) that duplicates the tier authority in another package without a consumer creates two sources of truth that will drift; either wire a consumer or have one map derive from the other.
- Status-transition handlers that unconditionally mutate side-effect fields (e.g., clearing `completedAt` on every non-`completed` transition) destroy history when the transition is to `archived`/`snoozed` rather than back to an active state.
