## Patterns
- When `@repo/ui` lacks a needed primitive (Checkbox, Select), keep the native element rather than blocking the migration; document the gap as follow-up.
- Wrap `@repo/ui` primitives with thin local components (`Label`, `Input`) to preserve the app's existing call sites while swapping the underlying implementation.
- Use inline `style` props on `Badge`/`Card` for per-instance theme color tokens (e.g. `c.color + "15"`) when the shared component's variant system doesn't cover dynamic colors.

- Deleting a review findings file (`review-issue-68.json`) without addressing or explicitly deferring its info-severity items leaves untracked debt; either file follow-up issues or note them in the PR.
