Shopping List — Documentation

Overview

A mobile-first shopping list app organized by store. Supports categories, drag-to-reorder, swipe-to-delete, quick-add suggestions, batch operations, and multiple sort options.

Stores

Three built-in stores: Trader Joe's, Costco, and Target. Each store has its own list with independent sort order.

Categories

Items can be categorized: Produce, Dairy, Meat & Seafood, Frozen, Bakery, Beverages, Snacks, Household, Personal Care, Other.

Data Schema

Table: shopping_items

id          TEXT PRIMARY KEY
store       TEXT NOT NULL
item_name   TEXT NOT NULL
quantity    INTEGER DEFAULT 1
category    TEXT DEFAULT 'other'
in_cart     BOOLEAN DEFAULT false
sort_order  INTEGER DEFAULT 0
created_at  TIMESTAMPTZ DEFAULT now()
updated_at  TIMESTAMPTZ DEFAULT now()

API (db.js)

ShoppingDB.init()

Initialize the database connection. Returns a ShoppingDB instance.

db.getItems(store)

Get all items for a store, sorted by sort_order then created_at.

db.addItem(store, name, qty, category)

Add an item. Auto-assigns next sort_order.

db.toggleInCart(id, inCart)

Toggle checked state.

db.updateSortOrder(id, order)

Update an item's position for drag-to-reorder.

db.moveToStore(id, newStore)

Move an item to a different store.

db.deleteItem(id) / db.deleteItems(ids)

Delete one or multiple items.

db.clearStore(store)

Delete all checked items from a store.

db.getRecentItems(limit)

Get unique recently-added item names for quick-add suggestions.

db.getStoreCounts()

Get item count per store for tab badges.

Features

  • Drag to reorder — drag grip handle in manual sort mode
  • Swipe to delete — swipe left/right on mobile to remove items
  • Quick-add pills — recently added items appear as one-tap buttons
  • Batch operations — long-press to enter select mode, bulk delete or move
  • Sort options — manual, name, category, date added, checked status
  • Store tab counts — item totals shown in tab labels
  • Confirmation modal — destructive actions use cc-modal, not confirm()
